# Unraid 升级后回归检查清单

适用场景：

- 已将镜像升级到新版本，例如 `icekale/stock-lu-tracker:v0.1.12`
- 已在 Unraid Compose Manager 或模板中完成拉取与重建
- 数据目录仍沿用原有挂载，例如 `/mnt/user/appdata/stock-lu/data`

建议按下面顺序检查，这样最容易快速定位问题。

## 1. 容器是否真的启动到新版本

在 Unraid 主机执行：

```bash
docker ps --filter name=stock-lu
docker logs --tail 50 stock-lu
curl http://127.0.0.1:8787/api/app-meta
```

预期结果：

- 容器状态为 `Up`
- 日志里没有持续重启、权限报错、端口占用报错
- `/api/app-meta` 返回的 `versionLabel` 为当前目标版本，例如 `v0.1.12`

如果版本不对：

- 检查 compose 或模板里的 `image` 标签是否已经更新
- 在 Unraid 中执行重新拉取并重建

## 2. 首页是否仍然公开可访问

浏览器打开：

- `http://<你的UnraidIP>:8787`

预期结果：

- 不需要输入后台密码
- 首页可以直接看到总览、持仓、历史月份、净值曲线
- 页脚版本号与刚部署的版本一致

如果页面样式不对或还是旧版本：

- 浏览器强制刷新一次
- 清空站点缓存后重试

## 3. 后台密码登录是否正常

浏览器打开：

- `http://<你的UnraidIP>:8787/admin-login.html`

预期结果：

- 输入后台密码后可进入后台配置页
- 登录后刷新 `admin.html` 不会立刻掉回登录页
- 首页仍保持公开访问

命令行快速检查：

```bash
curl -i http://127.0.0.1:8787/api/state
curl -i -H 'Content-Type: application/json' \
  -d '{"password":"你的后台密码"}' \
  http://127.0.0.1:8787/api/admin-auth/login
```

预期结果：

- 未登录访问受保护接口时返回 `401`
- 登录接口返回 `{"ok":true,"enabled":true}`

如果登录成功后还停留在登录页：

- 检查 `ADMIN_COOKIE_SECURE`
- 局域网 `http` 访问时应保持 `ADMIN_COOKIE_SECURE=false`
- 如果经由反向代理 `https` 访问，再改成 `true`

## 4. 配置保存是否正常

进入后台后检查：

- 雪球 Cookie
- 微博 Cookie
- OCR 配置
- 月度帖子目录相关配置

执行一次保存，预期结果：

- 页面提示保存成功
- 刷新后台页面后配置仍然存在

如果保存失败：

- 查看容器日志里是否有 `/app/data` 权限错误
- 检查挂载目录属主属组是否允许容器写入
- Unraid 常见做法是继续使用 `user: "99:100"` 或模板默认权限

## 5. 自动抓取与 OCR 是否正常

后台里执行一次：

- `立即抓取一次`

如果你启用了 OCR，再重点检查：

- 最新月度帖子是否被识别
- A 股和 H 股是否正确分开
- `00700.HK` 是否识别为“腾讯控股”
- 持仓金额、成本价、市值是否合理

建议重点核对一条已知样本：

- 雪球帖子：`https://xueqiu.com/8790885129/381996320`

预期结果：

- 运行日志中没有连续报错
- 最新持仓快照会更新
- 首页和后台中的持仓数据能对上截图内容

## 6. 月度帖子目录与批量导入是否正常

后台中执行：

- 打开“选取月度帖子目录”
- 测试全选、取消全选、单项勾选
- 导入 1 条已知月份记录

预期结果：

- 月度目录可以正常勾选
- 导入后首页可以切换到该月份
- 对应月份的持仓、历史记录、净值曲线都会刷新

如果目录为空：

- 检查雪球 Cookie 是否失效
- 检查标题正则是否仍匹配 `游戏仓YYYY年M月PS图`

## 7. 首页数据是否完整

升级后重点看这几项：

- 总市值
- 最新持仓
- 历史月份切换
- 月度净值曲线
- A 股 / H 股分区显示

预期结果：

- 切换月份后，持仓列表和历史记录同步变化
- 月度净值曲线没有明显断层或跳点异常
- 页脚显示当前版本

## 8. 权限与持久化是否正常

重启容器后再检查一次：

```bash
docker restart stock-lu
curl http://127.0.0.1:8787/api/app-meta
```

预期结果：

- 服务可正常重新起来
- 后台配置、历史月份、快照数据没有丢失

如果重启后数据丢失：

- 检查是否正确挂载到了 `/app/data`
- 确认不是把容器内临时目录当成了数据目录

## 9. 常见故障快速定位

### 9.1 页面还是旧版本

- 强制刷新浏览器
- 确认镜像标签已切到新版本
- 确认容器已经重新创建而不是只重启旧容器

### 9.2 后台保存失败

- 检查 `docker logs stock-lu`
- 重点看是否有 `EACCES`、`permission denied`、`store.json.tmp` 相关报错

### 9.3 登录成功但不能进入后台

- 检查 `ADMIN_COOKIE_SECURE`
- 检查浏览器是否拦截 Cookie
- 用 `http` 访问时不要开启 secure cookie

### 9.4 OCR 结果明显错误

- 优先检查 OCR 提供商配置是否生效
- 再核对帖子截图是否完整、是否被站点压缩
- 可先导入单月样本，确认再批量导入

## 10. 建议的最小回归组合

如果你每次升级后不想全量检查，至少做下面 6 项：

1. `curl /api/app-meta` 看版本号
2. 打开首页，看是否公开可访问
3. 登录后台，看是否能进入配置页
4. 保存一次配置，看是否持久化
5. 手动执行一次自动抓取
6. 导入或切换一个历史月份，确认首页联动正常
